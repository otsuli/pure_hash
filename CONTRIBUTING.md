# Contributing to pure_hash

Thanks for your interest in contributing to **pure_hash**.

This project is a small, performance-focused hashing library. Contributions are welcome as long as they keep the codebase simple, fast, and consistent.

---

## Code of Conduct

Be respectful. No toxic behavior, no spam, no ego wars. Keep discussions focused on improving the project.

---

## How to contribute

You can contribute in several ways:

- Bug fixes
- Performance improvements
- New hash algorithms (if justified)
- Documentation improvements
- Tests
- Build system improvements (CMake, LuaRocks packaging, etc.)

If you're unsure whether something fits, open an issue first.

---

## Before you start

1. Check existing issues and pull requests to avoid duplicates.
2. For major changes, open an issue first to discuss the idea.
3. Keep changes minimal and focused—don’t bundle unrelated stuff.
